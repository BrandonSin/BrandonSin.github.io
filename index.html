Front End Design for VR Viewing Platform


This was a project in my human computer interaction class. I worked with four other people including myself in trying to create a visual representation of how a VR viewing platform would look like. The goal of this project was to provide an interactive way for users to experience live events/recorded without having to be physically present. In doing so, we hope that this would be an incentive for musicians and artists to sell/promote their VR content using our application.




Connect Four


This was an individual project during my web application class. In this project I utilized socket.io to help create multiple game rooms and acknowledgments. On backend side I used express node.js and on the client side I used embedded js as the game was mainly done in javascript.Down below there are screenshots of the main menu and the in-game. The player has the ability to create a game, join a random game or enter a game using a passcode. An active player list is presented on the right of the main screen. All of these are done by the sockets communicating between the client and the server.





Crime Rate in Calgary

In this three person project (Jason Dam, Garland Khuu, Brandon Sin), we wanted to find a prevalent issue and visualize it. We came across a dataset found on Alberta Public Statistics that was composed of Crime statistics in Calgary during the years of 2012-2019. Our goal was to create a visualization that provides information corresponding to the user's interaction and using the LRT station as a landmark. Using the dataset, we extracted important data to create sketchable ideas for our final visualization. At the end, we decided to use a 2D map layout of Calgary and allow users to toggle between years, different layouts and clickable districts. Using the D3 library, we were able to create an interactive, aesthetically pleasing visual for any users to enjoy.
